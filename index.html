Web Development 9/3: Lists and Tables

Lists

  • [ol] tag is for ordered lists (html automatically orders them), use [li] for each list item
  • html automatically orders the list, and spaces them
  • [ul] tag is for unordered lists, and use same [li] tag for list items

Don't use tables to format what things look like (layout), problem is it doesn't adapt content to device you're on. Tables are good for data

Tables

  • [tr] tag defines table / [th] table header / [td] table data
  • Can use [a href] tag to add attributes
    "href" lets you link to a website/email, "target" lets you set where the link opens (e.g. if target is blank then it will open in a new tab)
    Example: [a href="__mailto:emailaddress__" target="_blank"][/a]
  • Can also wrap image in a link
    Example: [a href="___link___"][img src="__imagelink__"][/a]
  • [div] tag can be used to make meaning out of your elements/wrap elements (e.g. in a resume, have title and date separated with [div] tags, eventually can use CSS to control) - See coding in example below
